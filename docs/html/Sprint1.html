Requisiti del committente

Sprint0


L'obiettivo di questo Sprint consiste nella realizzazione e nell'integrazione dei componenti core-business del sistema. Al termine di questa fase di sviluppo sarà possibile consentire la registrazione dei prodotti con l'assegnazione di un identificativo univoco, la gestione di una richiesta di carico tenendo in considerazione i vincoli dati dal peso massimo e dalla disponibilità degli slot e infine la gestione dell'occupazione della stiva.

Requisiti dei contesti

Gestione Prodotto


ID Descrizione Tipo Priorità Vincoli
GP1 Il sistema deve permettere la registrazione di un nuovo prodotto, specificando almeno il peso Funzionale Alta Il peso deve essere espresso in kg, > 0
GP2 Alla registrazione di un prodotto, il sistema deve generare un identificativo univoco (PID) intero positivo. Funzionale Alta L'univocità del PID deve essere garantita
GP3 Il sistema deve consentire di ottenere il peso di un prodotto a partire dal PID Funzionale Alta
GP4 Il sistema deve mantenere le informazioni sui prodotti registrati per tutta la durata dell'esecuzione Funzionale Alta Persistenza delle informazioni in memoria
GP5 Il sistema può memorizzare anche informazioni opzionali sul prodotto Funzionale Bassa
GP6 Il PID viene generato in modo incrementale Non Funzionale Alta L'incremento deve essere unitario e costante per ogni nuovo prodotto registrato.

Gestione Prodotto


ID Descrizione Tipo Priorità Vincoli
GC1 Ricezione richiesta di carico per un PID già registrato Funzionale Alta PID valido (PID > 0 e già registrato)
GC2 Il sistema deve verificare che il peso massimo non venga superato Funzionale Alta Peso nuovo container + peso attuale ≤ MaxLoad, con MaxLoad configurabile e peso attuale ottenuto da InventarioStiva

GC4 Il sistema deve rifiutare una richiesta in caso di superamento peso massimo o slot non disponibile Funzionale Alta
GC5 Il sistema deve assegnare uno slot libero al PID in caso di richiesta valida Funzionale Alta Slot scelto restituito al chiamante
GC8 Il sistema deve interagire per garantire il corretto posizionamento del container nello slot assegnato Funzionale Alta
GC7 Il sistema deve bloccare l'elaborazione di nuove richieste fino al completamento della precedente Funzionale Media
